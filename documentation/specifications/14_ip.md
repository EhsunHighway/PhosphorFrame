# Module 14 - IPv4

**Files:** `src/protocols/ip.c`, `src/protocols/ip.h`
**Status:** Partial implementation with generic receive dispatch and direct-subnet output
**Depends on:** `byte_order`, `packet`, `interface`, `device`, `topology`,
`ethernet`, `arp_cache`, `arp`, `simulator`

## Concepts First

IPv4 is the layer between Ethernet and upper protocols such as ICMP, UDP, and
TCP.

Ethernet delivers frames on one local link. IPv4 delivers packets between IP
addresses.

In this simulator, IPv4 has two main directions:

```text
receive:
  Ethernet payload -> validate IPv4 header -> strip IPv4 header -> dispatch L4

send:
  L4 payload -> prepend IPv4 header -> resolve MAC -> Ethernet send
```

### IP Is A Layer Boundary

The IP module should know IPv4, not every protocol above it.

It should not include `icmp.h`, `udp.h`, or `tcp.h`.

Instead, upper protocols register handlers by protocol number:

```text
protocol 1  -> ICMP handler
protocol 6  -> TCP handler
protocol 17 -> UDP handler
```

When `ip_receive` validates and strips an IPv4 packet, it uses the protocol
number saved from the IPv4 header to call the registered handler.

### IpStack

`IpStack` is the per-owner IP dispatch table:

```c
typedef struct IpStack {
    Simulator       *sim;
    IpProtocolEntry  protocols[256];
} IpStack;
```

An interface is bound to an `IpStack` by installing `ip_rx_shim` as the
interface receive handler and storing the stack as `handler_ctx`.

```text
Interface.rx_handler  = ip_rx_shim
Interface.handler_ctx = IpStack *
```

The stack lets Host/Router code decide which upper protocols are enabled
without forcing IP to know their internal state.

### Receive Dispatch

Ethernet strips the Ethernet header first. Then it calls the interface receive
handler.

For IP-bound interfaces:

```text
ethernet_receive_event
  |
  +-- ethernet_receive(...)
  |
  +-- iface->rx_handler(iface, frame, ethertype, iface->handler_ctx)
        |
        +-- ip_rx_shim(...)
              |
              +-- ip_receive(iface, frame, ethertype, stack)
```

At entry to `ip_receive`, `frame->data` points at the IPv4 header.

On successful receive, `ip_receive` strips 20 bytes and the packet becomes a
layer-4 payload.

### Send Boundary: `ip_send` Versus `ip_output`

There are two send APIs:

```text
ip_send:
  caller already knows outgoing interface and destination MAC

ip_output:
  caller knows source IP, destination IP, protocol, and payload
  IP finds interface and resolves MAC
```

Upper protocols should normally call `ip_output`, not `ip_send`.

`ip_send` is a lower-level helper used after link-layer resolution is already
done, including ARP pending flush.

### Current Routing Scope

Current `ip_output` only handles direct-subnet destinations.

It does not yet do:

- route-table lookup
- default gateway
- next-hop selection
- router forwarding
- TTL decrement forwarding
- fragmentation
- ICMP unreachable/time-exceeded generation

For now:

```text
destination on same subnet -> ARP lookup/request -> send or enqueue
destination off subnet     -> return -1
```

The route table module will later extend this boundary.

### Byte Order

IPv4 header fields are wire fields, so multi-byte values in `IpHeader` are
network byte order.

Current public conventions:

- `ip_send` parameters `src_ip` and `dst_ip` are host order.
- `ip_output` parameters `src_ip` and `dst_ip` are host order.
- `Interface.ip_addr` is stored in network byte order.
- `ip_find_source_iface` compares `iface->ip_addr` to `ns_htonl(src_ip)`.
- `ip_output` calls `arp_cache_lookup` with host-order `dst_ip`.
- `ip_output` calls `arp_send_request` with `ns_htonl(dst_ip)`.
- `ip_is_local` expects `dst_ip` in the same byte order as `iface->ip_addr`,
  which is network order in current interface objects.

These conventions are easy to mix up. Keep them explicit in tests.

## Purpose

The IPv4 module validates, constructs, and dispatches IPv4 packets.

It provides:

- IPv4 header layout and constants
- IP stack initialization
- binding interfaces to an IP stack
- registering/unregistering protocol handlers
- receive validation and layer-4 dispatch
- direct-subnet output path
- lower-level send helper
- IPv4 header checksum helper
- local-address helper

It does not:

- own interfaces
- own ARP cache storage
- implement route-table lookup yet
- forward router transit packets yet
- fragment packets
- generate ICMP errors from IP failures yet
- know ICMP/UDP/TCP internal state

## Architecture Boundary

| Responsibility | Owner |
| --- | --- |
| Validate IPv4 header | IP |
| Strip IPv4 header on receive | IP |
| Dispatch by protocol number | IP stack |
| Implement ICMP/UDP/TCP receive behavior | Upper protocol modules |
| Build IPv4 header on send | IP |
| Resolve destination MAC on direct subnet | IP through ARP cache/ARP |
| Store ARP cache | Host/Router |
| Prepend Ethernet header and schedule link delivery | Ethernet/Link |
| Route off-subnet packets | Future route table/router path |

IP should remain generic. Registering ICMP/UDP/TCP handlers belongs to the stack
owner, such as Host or Router setup.

## Data Model

### Constants

```c
#define IP_HDR_LEN         20
#define IP_VERSION         4
#define IPPROTO_ICMP       1
#define IPPROTO_TCP        6
#define IPPROTO_UDP        17
#define IP_DEFAULT_TTL     64
#define IP_FLAG_DF         0x4000
#define IP_MAX_PACKET_SIZE 65535
```

### `IpProtocolHandler`

```c
typedef int (*IpProtocolHandler)(Interface *iface,
                                 Packet    *pkt,
                                 void      *ctx);
```

The handler receives:

- the interface that received the IPv4 packet
- the packet after the IPv4 header has been stripped
- the context registered for that protocol

### `IpProtocolEntry`

```c
typedef struct IpProtocolEntry {
    IpProtocolHandler handler;
    void             *ctx;
} IpProtocolEntry;
```

### `IpStack`

```c
typedef struct IpStack {
    Simulator       *sim;
    IpProtocolEntry  protocols[256];
} IpStack;
```

`protocols[protocol].handler == NULL` means no handler is registered for that
IPv4 protocol number.

### `IpHeader`

```c
typedef struct __attribute__((packed)) IpHeader {
    uint8_t  version_ihl;
    uint8_t  dscp_ecn;
    uint16_t total_length;
    uint16_t identification;
    uint16_t flags_fragment_offset;
    uint8_t  ttl;
    uint8_t  protocol;
    uint16_t header_checksum;
    uint32_t src_ip;
    uint32_t dst_ip;
} IpHeader;
```

Current supported header:

- IPv4 only
- minimum 20-byte header
- no IP options
- no fragmentation support

Send fills:

- `version_ihl = 0x45`
- `dscp_ecn = 0`
- `total_length = ns_htons(IP_HDR_LEN + payload->len)`
- `identification = 0`
- `flags_fragment_offset = 0`
- `ttl = 64`
- `protocol = caller protocol`
- `src_ip = ns_htonl(src_ip)`
- `dst_ip = ns_htonl(dst_ip)`
- checksum after zeroing `header_checksum`

## Ownership And Lifetime

The IP module does not allocate `IpStack`; the stack owner provides storage.

`ip_stack_init` initializes existing `IpStack` storage.

`ip_receive` mutates the received `Packet` in place by stripping the IPv4
header. It may free malformed packets on some failure paths.

Current receive ownership asymmetry:

- null or too-short receive returns `-1` without freeing the packet
- bad version, TTL zero, and bad checksum free the packet

That is current code behavior, not an ideal final ownership rule.

`ip_send` and `ip_output` do not clone payloads. On ARP pending enqueue success,
the pending queue owns the payload. On direct send, ownership follows the
Ethernet/link behavior.

Current lower-layer ownership caveat: `ethernet_send`/`link_transmit` clone for
scheduled delivery and do not free the caller packet, so direct-send ownership
still needs a project-wide rule.

## Function Behavior

### `ip_init`

Required behavior:

- If `sim == NULL`, return immediately.
- If `sim->topo == NULL`, return immediately.
- If `stack == NULL`, return immediately.
- Initialize `stack` with `ip_stack_init(stack, sim)`.
- For every device in `sim->topo->devices[0 .. dev_count - 1]`:
  - for every interface in `dev->interfaces[0 .. iface_count - 1]`:
    - bind that interface to `stack` with `ip_stack_bind_interface`

Current implementation assumes live topology devices are non-NULL before
reading `dev->iface_count`.

### `ip_stack_init`

Required behavior:

- If `stack == NULL`, return immediately.
- Clear the whole stack.
- Set `stack->sim = sim`.
- Leave every protocol handler/context slot zeroed.

### `ip_stack_bind_interface`

Required behavior:

- If `stack == NULL`, return `-1`.
- If `iface == NULL`, return `-1`.
- Set the interface receive handler to the internal `ip_rx_shim`.
- Set the interface handler context to `stack`.
- Return `0`.

### `ip_stack_register_protocol`

Required behavior:

- If `stack == NULL`, return `-1`.
- If `handler == NULL`, return `-1`.
- Store `handler` in `stack->protocols[protocol].handler`.
- Store `ctx` in `stack->protocols[protocol].ctx`.
- Return `0`.

All `uint8_t` protocol values are valid array indexes `0..255`.

### `ip_stack_unregister_protocol`

Required behavior:

- If `stack == NULL`, return `-1`.
- Set `stack->protocols[protocol].handler = NULL`.
- Set `stack->protocols[protocol].ctx = NULL`.
- Return `0`.

### `ip_receive`

Required behavior:

- Ignore `ethertype`.
- If `iface == NULL`, return `-1`.
- If `frame == NULL`, return `-1`.
- If `frame->len < IP_HDR_LEN`, return `-1`.
- Interpret `frame->data` as `IpHeader`.
- If version nibble is not `4`:
  - increment `iface->rx_errors`
  - free `frame`
  - return `-1`
- If `ttl == 0`:
  - increment `iface->rx_dropped`
  - free `frame`
  - return `-1`
- If `ip_checksum(ip_hdr) != 0`:
  - increment `iface->rx_errors`
  - free `frame`
  - return `-1`
- Save `protocol = ip_hdr->protocol`.
- Add the original visible IPv4 packet length to `iface->rx_bytes`.
- Interpret `ctx` as `IpStack *`.
- If stack and stack simulator exist, set `iface->last_rx_time` to
  `simulator_now(stack->sim)`.
- Otherwise set `iface->last_rx_time = 0`.
- Advance `frame->data` by `IP_HDR_LEN`.
- Decrease `frame->len` by `IP_HDR_LEN`.
- Set `frame->layer = 4`.
- If no stack exists, return `0`.
- Look up `stack->protocols[protocol]`.
- If a handler exists, call it and return its result.
- If no handler exists, return `0`.

Current implementation does not verify:

- `version_ihl` IHL nibble is exactly `5`
- `total_length` matches packet length
- destination IP is local
- fragmentation flags/offset

### `ip_send`

Required behavior:

- If `sim == NULL`, return `-1`.
- If `iface == NULL`, return `-1`.
- If `dst_mac == NULL`, return `-1`.
- If `payload == NULL`, return `-1`.
- Build a stack-local `IpHeader`.
- Fill the IPv4 header fields described in Data Model.
- Compute header checksum after zeroing checksum field.
- Prepend the IPv4 header into `payload`.
- If prepend fails, return `-1`.
- Set `payload->layer = 3`.
- Add the IPv4 packet length to `iface->tx_bytes`.
- Set `iface->last_tx_time = simulator_now(sim)`.
- Call:

```c
ethernet_send(sim, iface, dst_mac, ETHERTYPE_IPV4, payload)
```

- Return Ethernet's result.

Because `ethernet_send` currently returns `1` for successful scheduled link
delivery, `ip_send` may return `1` on success.

### `ip_output`

Required behavior:

- If `sim == NULL`, return `-1`.
- If `payload == NULL`, return `-1`.
- Find the source interface by scanning topology for
  `iface->ip_addr == ns_htonl(src_ip)`.
- If no source interface exists, return `-1`.
- If source interface is down, return `-1`.
- If source interface has no ARP cache, return `-1`.
- Convert `iface->ip_addr` to host order.
- Check whether `dst_ip` is on the same subnet as the source interface.
- If destination is not on the same subnet, return `-1`.
- Look up `dst_ip` in `iface->arp_cache`.
- If ARP lookup succeeds, call `ip_send`.
- If ARP lookup fails:
  - send ARP request for `ns_htonl(dst_ip)`
  - if ARP request fails, return `-1`
  - enqueue payload into ARP pending queue with host-order metadata
  - if enqueue fails, return `-1`
  - return `0`

On ARP pending enqueue success, payload ownership moves to the pending queue.

### `ip_checksum`

Required behavior:

- If `ip_hdr == NULL`, return `0xFFFF`.
- Treat the fixed 20-byte IPv4 header as ten 16-bit words.
- Convert each word from network order to host order with `ns_ntohs`.
- Sum the words.
- Fold carries into the low 16 bits.
- Return the one's complement in network byte order with `ns_htons`.

When validating a header whose checksum field is already filled, a valid header
should make `ip_checksum(ip_hdr) == 0`.

### `ip_is_local`

Required behavior:

- If `dev == NULL`, return `0`.
- Scan live interfaces from index `0` to `iface_count - 1`.
- If any interface has `ip_addr == dst_ip`, return `1`.
- Otherwise return `0`.

`dst_ip` must be in the same byte order as `Interface.ip_addr`.

## Flow Charts

### Receive

```text
ip_receive(iface, frame, ethertype, stack)
  |
  +-- reject NULL or too-short packet
  |
  +-- ip_hdr = frame->data
  |
  +-- version is not IPv4:
  |     rx_errors++, free frame, return -1
  |
  +-- ttl == 0:
  |     rx_dropped++, free frame, return -1
  |
  +-- checksum bad:
  |     rx_errors++, free frame, return -1
  |
  +-- save protocol
  |
  +-- rx_bytes += original IPv4 length
  +-- last_rx_time = stack sim time or 0
  |
  +-- strip IPv4 header
  +-- frame->layer = 4
  |
  +-- no stack: return 0
  |
  +-- registered handler for protocol?
        |
        +-- yes: return handler(iface, frame, handler_ctx)
        |
        +-- no: return 0
```

### Direct Output

```text
ip_output(sim, src_ip, dst_ip, protocol, payload)
  |
  +-- reject NULL sim or payload
  |
  +-- find interface whose ip_addr == htonl(src_ip)
  |
  +-- reject missing/down/no-arp-cache interface
  |
  +-- reject destination not on same subnet
  |
  +-- arp_cache_lookup(dst_ip)
        |
        +-- hit:
        |     ip_send(...)
        |
        +-- miss:
              arp_send_request(... htonl(dst_ip))
              arp_pending_enqueue(...)
              return 0 on enqueue success
```

### Lower-Level Send

```text
ip_send(sim, iface, dst_mac, src_ip, dst_ip, protocol, payload)
  |
  +-- reject NULL inputs
  |
  +-- build 20-byte IPv4 header
  +-- compute checksum
  |
  +-- packet_prepend(payload, &ip_hdr, 20)
  |
  +-- payload->layer = 3
  +-- tx_bytes += payload->len
  +-- last_tx_time = simulator_now(sim)
  |
  +-- ethernet_send(... ETHERTYPE_IPV4 ...)
  |
  +-- return Ethernet result
```

## ACSL Contracts

The contracts belong in `ip.h`. Keep IP generic: do not mention concrete ICMP,
UDP, or TCP types in IP ACSL.

### Shared Predicates

```c

    predicate ip_stack_valid(IpStack *stack) =
        \valid(stack);

    disjoint behaviors;
*/
int ip_is_local(Device *dev, uint32_t dst_ip);
```

Additional required proof/test property:

- Returns `1` exactly when a live interface has `ip_addr == dst_ip`.

## Common Mistakes

- Do not include ICMP, UDP, or TCP headers in IP just to dispatch protocols.
- Do not pass network-order source/destination IPs to `ip_send` or `ip_output`.
- Do not pass host-order IP to `ip_is_local`; it compares against interface
  stored order.
- Do not describe route-table lookup as implemented yet.
- Do not forget that ARP miss success returns `0` because payload is queued.
- Do not hide current receive failure ownership asymmetry.
- Do not validate checksum by checking the checksum field against `0xFFFF`;
  current code calls `ip_checksum(ip_hdr) != 0`.
