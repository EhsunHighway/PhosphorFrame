# Module 17 - Host

**Files:** `src/network/host.c`, `src/network/host.h`  
**Status:** Ready for implementation  
**Depends on:** `device`, `interface`, `arp_cache`, `arp`, `ip`, `icmp`, `udp`, `tcp`, `simulator`

## Purpose

A Host is an endpoint in the simulator. It is the object that turns the lower
network pieces into one usable machine:

- it owns a `Device` base so it can have interfaces and links
- it owns one ARP cache for resolving neighbors on its local networks
- it owns one IP stack for receiving IPv4 packets and dispatching transport
  protocols
- it owns one UDP state table
- it owns one TCP connection table
- it stores an optional default gateway address for future off-subnet routing

A Host is not a switch and not a router. It does not forward packets between
interfaces. It sends packets generated by local protocols and receives packets
addressed to one of its local interfaces.

## Mental Model

Think of Host as the owner of per-machine protocol state.

```text
incoming frame
    |
    v
Interface rx_handler
    |
    v
ip_receive using this Host's IpStack
    |
    +--> ICMP handler
    +--> UDP handler using this Host's UdpState
    +--> TCP handler using this Host's TcpTable

local application or protocol send
    |
    v
UDP/TCP/ICMP creates payload
    |
    v
ip_output chooses interface and resolves ARP
    |
    v
Ethernet transmit
```

The important boundary is this: UDP and TCP should not use one global state
table for the whole simulator. Each Host has its own UDP and TCP state, and the
Host registers those states with its own `IpStack`.

## Data Ownership

The Host owns:

- the embedded `Device` storage
- the array of interface pointers used by that device
- the interfaces added to it, if the Host is responsible for freeing the device
  like other `Device` owners
- the `ArpCache`
- the `IpStack`
- the `UdpState`
- the `TcpTable`
- the small UDP/TCP context structs passed to IP dispatch

The Host does not own:

- packets after a successful `ip_output`
- packets passed into receive handlers after ownership is transferred to a
  protocol handler
- the simulator itself

## Host Structure

`Host` should embed `Device` as its first field. This lets existing code treat a
`Host *` as a `Device *` when connecting links or adding interfaces.

```c
typedef struct Host {
    Device     base;

    Simulator *sim;

    ArpCache   arp_cache;
    IpStack    ip_stack;

    UdpState   udp_state;
    UdpContext udp_ctx;

    TcpTable   tcp_table;
    TcpContext tcp_ctx;

    uint32_t   gateway_ip;
} Host;
```

`gateway_ip` is stored in host byte order. `0` means no default gateway is
configured.

The interface field `iface->ip_addr` is still stored in network byte order,
because that is how the existing interface and IP code represent configured
interface addresses.

## Public API

The Host module should expose a small API that creates a machine, attaches
interfaces, and provides optional convenience send/receive entry points.

Suggested declarations:

```c
Host *host_create(const char *name, Simulator *sim, uint32_t gateway_ip);
void  host_free(Host *host);

int   host_add_interface(Host *host, Interface *iface);

int   host_receive(Host *host,
                   Interface *iface,
                   Packet *pkt,
                   uint16_t ethertype);

int   host_send_ip(Host *host,
                   uint32_t src_ip,
                   uint32_t dst_ip,
                   uint8_t protocol,
                   Packet *payload);
```

Higher-level application helpers can be added later, but UDP and TCP already
provide their own protocol send functions. Host should not duplicate full UDP or
TCP packet construction.

## Byte Order Rules

Use these rules consistently:

- `gateway_ip` is host order
- `host_send_ip` `src_ip` and `dst_ip` are host order
- UDP/TCP public API IP arguments are host order
- ARP cache lookup keys are host order in the current stack
- `Interface.ip_addr` is network order
- wire headers are network order

Host mostly connects modules together, so its main byte-order job is to avoid
rewriting addresses unnecessarily. Let `ip_output`, `ip_send`, UDP, TCP, ICMP,
and ARP write their own wire headers.

## `host_create`

Create one Host and initialize all per-host state.

Behavior:

- If `name` is `NULL`, return `NULL`.
- If `sim` is `NULL`, return `NULL`.
- Allocate and zero one `Host`.
- Initialize the embedded `Device` fields:
  - copy `name` into the device name buffer
  - allocate the interface pointer array with capacity `HOST_MAX_PORTS`
  - set the interface count to zero
  - set the device kind/type consistently with the rest of the project, if the
    device module has such a field
- Initialize the ARP cache as empty.
- Initialize the IP stack with `ip_stack_init(&host->ip_stack, sim)`.
- Initialize UDP with `udp_init(&host->udp_state)`.
- Initialize TCP with `tcp_init(&host->tcp_table)`.
- Fill `host->udp_ctx` with the simulator and the host's UDP state.
- Fill `host->tcp_ctx` with the simulator and the host's TCP table.
- Register IP protocol handlers:
  - protocol `IPPROTO_ICMP` uses `icmp_receive` with `sim` as context
  - protocol `IPPROTO_UDP` uses `udp_receive` with `&host->udp_ctx`
  - protocol `IPPROTO_TCP` uses `tcp_receive` with `&host->tcp_ctx`
- Store `gateway_ip`.
- Return the Host.

If any allocation fails, release everything already allocated and return `NULL`.

## `host_free`

Release all memory owned by the Host.

Behavior:

- If `host` is `NULL`, return.
- Free interfaces owned through the embedded device, following the same
  ownership rule as `device_free`.
- Free the embedded device's interface pointer array.
- Free the Host itself.

No special cleanup is needed for `ArpCache`, `IpStack`, `UdpState`, or
`TcpTable` in this phase because they are embedded fixed-size state. If future
versions add heap-owned queues or application objects, Host cleanup must release
those before freeing the Host.

## `host_add_interface`

Attach an interface to a Host and bind it into the host's network stack.

Behavior:

- If `host` is `NULL`, return `-1`.
- If `iface` is `NULL`, return `-1`.
- Add the interface to the embedded device with `device_add_interface`.
- If the device is full or rejects the interface, return `-1`.
- Set the interface ARP cache to the host-owned cache with
  `interface_set_arp_cache(iface, &host->arp_cache)`.
- Bind the interface to the host IP stack with
  `ip_stack_bind_interface(&host->ip_stack, iface)`.
- Return `0` on success.

The interface does not need to be administratively up when it is added. The
send/receive paths already check interface state where it matters.

## `host_receive`

`host_receive` is a convenience wrapper for delivering an incoming frame to the
host's IP stack. It is useful in tests or in code that wants to enter at the Host
boundary instead of calling the interface handler directly.

Behavior:

- If `host`, `iface`, or `pkt` is `NULL`, return `-1`.
- If `ethertype` is not IPv4, free `pkt` and return `-1`.
- Call `ip_receive(iface, pkt, ethertype, &host->ip_stack)`.
- Return the result from `ip_receive`.

Normal link delivery may call the interface receive handler directly. That is
still fine, because `host_add_interface` binds each interface to the Host's
`IpStack`.

## `host_send_ip`

Send an already-built layer-4 payload through the Host.

Behavior:

- If `host` is `NULL`, return `-1`.
- If `payload` is `NULL`, return `-1`.
- If `src_ip` is `0`, return `-1`.
- If `dst_ip` is `0`, return `-1`.
- Call `ip_output(host->sim, src_ip, dst_ip, protocol, payload)`.
- Return the result from `ip_output`.

Ownership follows `ip_output`:

- on success, the payload is owned by IP, ARP pending state, or the lower layer
- on failure, the caller still owns the payload and must free it

`host_send_ip` should not manually choose a destination MAC address and should
not perform ARP lookup itself. That belongs in `ip_output`.

## Gateway And Routing Boundary

The Host stores `gateway_ip`, but Phase 1 does not implement gateway routing in
Host.

Current `ip_output` sends only to destinations reachable on the same subnet as
the chosen source interface. If the destination is off-subnet, `ip_output`
returns `-1`.

Future gateway support should be added by extending IP output/routing behavior,
not by making Host manually bypass IP. A clean future rule is:

- Host owns the configured gateway value.
- IP output decides whether the next hop is the destination or the gateway.
- ARP resolves the selected next-hop address.

## ACSL Highlights

The header contracts should focus on ownership, null behavior, and observable
state. Keep them readable; this module is mostly orchestration.

`host_create`:

- `name == NULL || sim == NULL` returns `NULL`
- valid inputs return either `NULL` on allocation failure or a valid Host
- success initializes per-host protocol state

`host_add_interface`:

- `host == NULL || iface == NULL` returns `-1`
- success adds one interface, sets `iface->device`, sets `iface->arp_cache`, and
  installs IP receive dispatch on the interface

`host_receive`:

- null Host/interface/packet returns `-1`
- non-IPv4 frees the packet and returns `-1`
- IPv4 delegates to `ip_receive`

`host_send_ip`:

- null Host/payload returns `-1`
- zero source or destination address returns `-1`
- valid input delegates to `ip_output`

Use literal numeric bounds in ACSL where the prover has trouble with macros.
For example, use `8` instead of `HOST_MAX_PORTS` in quantified ranges if needed.

## Tests

Minimum useful tests:

1. `host_create(NULL, sim, 0)` returns `NULL`.
2. `host_create("h1", NULL, 0)` returns `NULL`.
3. A valid Host starts with zero interfaces, an empty ARP cache, an initialized
   UDP state, and an initialized TCP table.
4. `host_add_interface(NULL, iface)` returns `-1`.
5. `host_add_interface(host, NULL)` returns `-1`.
6. `host_add_interface(host, iface)` sets `iface->device`, `iface->arp_cache`,
   `iface->rx_handler`, and `iface->handler_ctx`.
7. `host_receive` rejects non-IPv4 ethertypes and consumes the packet.
8. `host_send_ip` rejects null payload and zero addresses.
9. With a same-subnet destination and an ARP hit, `host_send_ip` reaches
   `ip_output` successfully.
10. With a same-subnet destination and an ARP miss, `host_send_ip` lets
    `ip_output` queue the payload and send ARP.

## Common Mistakes

- Do not use one global UDP or TCP state for every Host.
- Do not store host-order addresses directly in wire headers.
- Do not make Host perform Ethernet destination MAC selection.
- Do not make Host forward packets like a router.
- Do not call `device_create` and then separately allocate Host unless you have
  a clear ownership wrapper. The clean design is one allocated Host with an
  embedded `Device` as the first field.
- Do not free a payload inside `host_send_ip` when `ip_output` fails; the caller
  still owns it.
